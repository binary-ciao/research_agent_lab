ExperimentOrchestratorAgent 内部循环

Workflow 视角只看到一个 agent，内部封装完整的 code → run → debug → retry 循环

ExperimentOrchestratorAgent.run() 流程图
1. 初始化
读取 state.values["experiment_plans"], ["code_tasks"]
attempt = 0, max_attempts = 3
all_patches = [], all_results = [], all_debug_records = []
↓
对每个 ExperimentPlan 执行:
△ 子循环 (per plan): while attempt ≤ max_attempts
Step A: CodeWriterAgent.run(state, context)
→ 首次: 根据 ExperimentPlan 生成代码
→ retry: 应用 AutoDebugger 的修复代码
→ 输出: CodePatch → state.values["code_patch"]
→ artifact: code_patches/{patch_id}.json
↓
Step B: AutonomousExperimentAgent.run(state, context)
→ 在 work_dir 运行实验命令
→ parse_experiment_output() + _check_criteria()
→ 输出: ExperimentResult[] → state.values["experiment_results"]
→ artifact: experiment_results/{result_id}.json
↓
Step C: 判断
✅ 所有结果 passed → 记录成功，break（进入下一个 plan）
❌ 全部失败 且 attempt ≥ max → 记录失败，break
⚠️ 部分失败且 attempt < max → 继续 Step D
↓ retry
Step D: AutoDebuggerAgent.run(state, context)
→ 解析 traceback，定位错误文件
→ LLM 分析 + 生成修复代码
→ 输出: AutoDebugRecord → state.values["last_debug_record"]
→ artifact: auto_debug_records/{record_id}.json
→ attempt += 1，回到 Step A
↓ 所有 plan 处理完
2. 汇总输出
state.values["all_code_patches"] = all_patches
state.values["all_experiment_results"] = all_results
state.values["experiment_orchestrator_summary"] = "{n} plans, {p} passed, {f} failed, {d} debug records"

关键规则

子 agent 调用直接实例化并调 run(state, context)，复用同一个 state 和 context
Artifact每个子 agent 独立保存 artifact，和 workflow 中其他 agent 一致
LLM budget每次调用前检查 llm_budget_allows()；超预算时 CodeWriter/AutoDebugger 返回 skipped
多 plan 处理按 plans 顺序串行，前一个 plan 失败不阻止后续 plan
workflow 位置DeveloperAgent 之后，ExperimentDecisionAgent 之前
