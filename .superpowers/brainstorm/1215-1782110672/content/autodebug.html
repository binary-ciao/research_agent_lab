Auto-Debug 修复策略

实验运行失败或结果不达标时，agent 如何诊断并修复？

A

规则驱动修复

解析 stderr/stdout 中的常见错误模式（ImportError、SyntaxError、AssertionError、command not found 等），匹配预定义的修复模板。无 LLM 调用

优点

  • 零 token 消耗
  • 快速，确定性
  • 不会越改越错

缺点

  • 只能修常见错误
  • 逻辑 bug 无法处理
  • 维护规则库成本
B

LLM 驱动修复（AI Scientist-v2 风格）

把失败日志 + 原始代码 + 实验计划一起发给 LLM，让 LLM 分析原因并生成 patch。最多 3 轮修复

优点

  • 可处理复杂逻辑 bug
  • 自适应，无需规则库
  • 与 AI Scientist-v2 一致

缺点

  • 消耗 token
  • 可能引入新 bug
  • 需要 flash 模型路由
C

分级修复（推荐）

第一轮：规则驱动快速修复（ImportError → pip install, SyntaxError → 修语法等）。规则无法匹配时，升级到 LLM 修复（最多 2 轮）。总共不超过 3 轮

优点

  • 简单问题不浪费 token
  • 复杂问题有 LLM 兜底
  • 轮次可控

缺点

  • 需要同时实现两种
  • 升级判断可能不准确
