实验执行链路架构（方案 A）

外循环：每个 agent 独立运行，artifact 可审计；内循环：CodeWriter ↔ AutoDebugger 最多 3 轮修复

整体数据流

Experiment Execution Pipeline
1. ExperimentPlanner (已有，增强)
输入: synthesis + method_cards + codebase_report
输出: ExperimentPlan (新增 success_criteria 字段)
↓
2. DeveloperAgent (已有，保留)
输入: ExperimentPlan
输出: CodeTask (implementation_notes, proposed_files)
↓
3. CodeWriterAgent NEW
输入: CodeTask + ExperimentPlan + TopicPack.codebase
操作: 副本/沙箱模式下编辑代码文件
输出: CodePatch (diff, changed_files, backup_paths)
↓
4. AutonomousExperimentAgent (已有，增强)
输入: CodePatch + ExperimentPlan.commands
操作: 在代码副本/沙箱中运行实验命令
输出: ExperimentRun (stdout, stderr, exit_code, duration, metrics)
↓
5. ResultParserAgent NEW
输入: ExperimentRun + ExperimentPlan.success_criteria
操作: 正则提取指标 → 与目标对比；无指标时 LLM Judge
输出: ParseResult (status: pass/fail, parsed_metrics, judge_reason)
↓
✅ pass
→ ExperimentDecisionAgent
❌ fail → AutoDebuggerAgent NEW
失败日志 + 代码 + 计划 → LLM 分析 → 生成修复 patch
→ 回到 CodeWriterAgent（应用修复，最多 3 轮）
↓ pass
6. ExperimentDecisionAgent (已有，增强)
输入: ParseResult + 历史实验记录
输出: decision (investigate / continue / rollback / hold)

与现有 P10 实验树的集成

ExperimentDecisionAgent 的 continue 决策 → TreeSearchAgent 创建新分支节点 → BranchSelectionAgent 选择下一批实验 → 循环

关键设计约束

修复轮次上限3 轮（AutoDebugger → CodeWriter → Experiment → ResultParser）
安全模型默认代码副本；codebase.copy_can_modify=true 时沙箱模式
代码变更范围限制在 allowed_paths 白名单，排除 protected_paths
LLM 路由CodeWriter 用 experiment_planner 路由；AutoDebugger 用 paper_triage flash 路由
每步 artifactcode_patches/、experiment_runs/、parse_results/、auto_debug_records/
