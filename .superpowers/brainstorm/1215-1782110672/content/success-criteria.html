实验成功判定方式

怎么判断一个实验"成功了"——决定了 debug 循环何时停止、何时推进到下一阶段

A

指标解析 + 目标对比（AI Scientist-v2 风格）

ExperimentPlan 中预先定义目标指标（如 accuracy >= 0.9）。ResultParser 从 stdout 正则提取数值，自动对比判定 pass/fail

优点

  • 全自动，不需要 LLM
  • 可量化，可重现
  • 与 AI Scientist-v2 一致

缺点

  • 依赖 stdout 格式规范
  • 无法判断定性结果（如图像质量）
B

LLM Judge 判定

将实验 stdout + stderr + 预期目标发给 LLM，让 LLM 判断是否达到目标并输出理由

优点

  • 可处理定性目标
  • 灵活，适应各种输出格式

缺点

  • 消耗 token
  • 判定可能不一致
  • 每次 run 结果可能不同
C

双层判定（推荐）

优先指标解析（快速、零 token）。如果 ExperimentPlan 中定义了数值指标就用它；如果只有定性目标（"改善生成图像的多样性"）或指标解析失败，fallback 到 LLM Judge

优点

  • 能定量的不浪费 token
  • 定性的也能处理
  • 渐进式能力

缺点

  • 两种实现
